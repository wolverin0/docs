## Convencion Feature-Folder

Cada dominio de negocio sigue esta estructura interna:

```
src/features/{domain}/
├── components/     # Componentes React del dominio
│   ├── {entity}-list.tsx
│   ├── {entity}-form.tsx
│   └── {entity}-detail.tsx
├── hooks/          # Hooks de TanStack Query
│   └── use-{entity}.ts
├── lib/            # Schemas, constantes, columnas, utilidades
│   ├── {entity}-schema.ts
│   ├── {entity}-constants.ts
│   └── {entity}-columns.tsx
└── types.ts        # Tipos TypeScript del dominio
```

## Patron de rutas (TanStack Router)

<Warning>
  TanStack Router usa **layout + index** para rutas anidadas. **Nunca renderices un componente directamente en el archivo layout.** Siempre usa `Outlet` + archivo index separado.
</Warning>

```
_authenticated/
  eventos.tsx         # Layout: renderiza <Outlet />
  eventos.index.tsx   # Index: renderiza EventList
  eventos.nuevo.tsx   # Child: renderiza formulario de creacion
  eventos.$eventId.tsx # Child: renderiza detalle del evento
```

El archivo layout (`eventos.tsx`) solo contiene:

```tsx
import { Outlet, createFileRoute } from '@tanstack/react-router'

export const Route = createFileRoute('/_authenticated/eventos')({
  component: () => <Outlet />,
})
```

El archivo index (`eventos.index.tsx`) renderiza el componente principal:

```tsx
import { createFileRoute } from '@tanstack/react-router'
import { EventList } from '@/features/events/components/event-list'

export const Route = createFileRoute('/_authenticated/eventos/')({
  component: EventList,
})
```

## Patron de hooks (TanStack Query)

Todos los hooks de datos siguen este patron estandar:

```typescript
// use-{entity}.ts

// Query: lectura de datos
export function useEntities(filters?) {
  return useQuery({
    queryKey: ['entities', filters],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('entities')
        .select('*, related_table(id, name)')
        .order('created_at', { ascending: false })

      if (error) throw error
      return data
    },
    staleTime: 10 * 60 * 1000, // 10 minutos
  })
}

// Mutation: creacion
export function useCreateEntity() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: async (data: EntityInsert) => {
      const { error } = await supabase
        .from('entities')
        .insert(data as any) // cast as any hasta generar tipos

      if (error) throw error
    },
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['entities'] })
      toast.success('Entidad creada exitosamente')
    },
    onError: (error) => {
      toast.error(`Error al crear: ${error.message}`)
    },
  })
}
```

<Info>
  El `staleTime` de 10 minutos evita refetches innecesarios. Los datos se invalidan explicitamente al crear/editar/eliminar.
</Info>

## Patron de formularios

Los formularios usan `react-hook-form` + `zod` + shadcn/ui:

```typescript
// {entity}-schema.ts
import { z } from 'zod'

export const entitySchema = z.object({
  name: z.string().min(1, 'El nombre es requerido'),
  email: z.string().email('Email invalido').optional(),
  amount: z.number().min(0, 'El monto debe ser positivo'),
})

export type EntityFormValues = z.infer<typeof entitySchema>
```

```tsx
// {entity}-form.tsx
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { entitySchema, type EntityFormValues } from './entity-schema'

export function EntityForm() {
  const form = useForm<EntityFormValues>({
    resolver: zodResolver(entitySchema),
    mode: 'onSubmit', // para performance
    defaultValues: { name: '', amount: 0 },
  })

  // ... render con shadcn/ui Form components
}
```

## Definicion de columnas

Las columnas de tablas se definen en archivos separados para reutilizacion:

```tsx
// lib/{entity}-columns.tsx
import type { ColumnDef } from '@tanstack/react-table'
import type { Entity } from '../types'
import { formatCurrency } from '@/lib/format'

export const entityColumns: ColumnDef<Entity>[] = [
  {
    accessorKey: 'name',
    header: 'Nombre',
  },
  {
    accessorKey: 'amount',
    header: 'Monto',
    cell: ({ row }) => formatCurrency(row.original.amount),
  },
]
```
