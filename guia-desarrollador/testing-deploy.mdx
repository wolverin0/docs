# Testing y Deploy

## Tests E2E (Playwright)

El proyecto cuenta con 26 tests E2E que cubren todos los flujos criticos de la aplicacion.

### Comandos

| Comando | Descripcion |
|---------|-------------|
| `npx playwright test` | Ejecutar todos los tests (headless, ~90 segundos) |
| `npx playwright test --headed` | Ejecutar con navegador visible |
| `npx playwright test -g "Payments"` | Ejecutar solo tests que matcheen el nombre |
| `npx playwright test --ui` | Abrir el picker interactivo de Playwright |

### Cobertura de tests

Los 26 tests cubren:

| Area | Tests | Que verifica |
|------|-------|-------------|
| **Login** | 1 | Flujo de autenticacion con credenciales de dev |
| **Navegacion** | 8 | Los 8 links del sidebar cargan correctamente |
| **Dashboard** | 2 | KPI cards visibles, filtros de fecha funcionan |
| **Eventos** | 4 | Listado, creacion, detalle con 5 pestanas, accordion de costos |
| **Pagos** | 2 | Listado con filtros, formulario de creacion |
| **Proveedores** | 1 | Listado de proveedores |
| **Clientes** | 1 | Listado de clientes |
| **Exportacion** | 4 | Boton "Exportar" en las 4 paginas de listado |
| **Console errors** | 3 | Verificacion de 0 errores de consola en todas las paginas |

### Agregar nuevos tests

```typescript
// tests/e2e-full.spec.ts
test('should do something new', async ({ page }) => {
  // Helper de login reutilizable
  await login(page)

  // Navegar a la ruta
  await page.goto(`${BASE_URL}/your-route`)
  await page.waitForLoadState('networkidle')

  // Interactuar y verificar
  await expect(page.locator('text=Expected')).toBeVisible()

  // Screenshot para debugging
  await page.screenshot({ path: 'tests/screenshots/new-test.png' })
})
```

<Tip>
  Ejecuta los tests despues de cada cambio de codigo: `npx playwright test`. Los tests tardan aproximadamente 90 segundos en completarse.
</Tip>

### Cuando ejecutar tests

| Momento | Comando | Obligatorio |
|---------|---------|-------------|
| Despues de cualquier cambio | `npx playwright test` | Recomendado |
| Antes de commit | `npx playwright test` | Si |
| Despues de agregar features | Agregar tests + `npx playwright test` | Si |
| CI/CD (push) | Automatico via GitHub Actions | Si |

## Build y deploy

### Build de produccion

```bash
npm run build
```

Este comando ejecuta `tsc -b && vite build`:
1. **TypeScript**: Verifica tipos en todo el proyecto
2. **Vite**: Bundlea la aplicacion para produccion

<Warning>
  Si el build falla, no hagas push. Corrige los errores de TypeScript primero con `npx tsc --noEmit` para ver los errores detallados.
</Warning>

### Deploy a Vercel

<Steps>
  <Step title="Instalar Vercel CLI">
    ```bash
    npm i -g vercel
    ```
  </Step>
  <Step title="Deploy">
    ```bash
    npx vercel
    ```
  </Step>
  <Step title="Variables de entorno">
    Configurar en el dashboard de Vercel:
    - `VITE_SUPABASE_URL`
    - `VITE_SUPABASE_ANON_KEY`
  </Step>
</Steps>

### Deploy de migraciones

```bash
npx supabase db push --linked
```

Aplica todas las migraciones pendientes en el proyecto de Supabase vinculado.

### Deploy de Edge Functions

```bash
npx supabase functions deploy <function-name> --linked
```

Las 8 Edge Functions se despliegan individualmente: `afip-emit`, `create-user`, `invite-user`, `manage-user`, `mercadopago-checkout`, `mercadopago-webhook`, `send-email`, `workflow-execute`.

## Convenciones

### Idioma

- Toda la UI esta en **espanol**: labels, mensajes, tooltips, placeholders
- Los estados usan nombres en espanol: Borrador, En Curso, Completado, Cancelado, Pendiente, Pagado
- Los nombres de tablas y columnas en la base de datos mezclan espanol e ingles (legacy)

### Moneda

- Todos los montos son en **ARS** (peso argentino)
- Formato: `$1.234.567,89` via `formatCurrency()` de `src/lib/format.ts`
- Tipo de dato en DB: `NUMERIC(15,2)`, nunca floats de JavaScript

### Commits

- Formato: `feat(scope):`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`, `perf:`
- Commits atomicos: un cambio logico por commit
- Nunca commitear archivos `.env`, secretos ni `node_modules`

### Calidad de codigo

| Regla | Limite |
|-------|--------|
| Funciones | < 50 lineas |
| Archivos | < 800 lineas |
| Nesting | < 4 niveles |
| `console.log` en produccion | Prohibido |
| Secretos hardcodeados | Prohibido |
| Mutaciones | Usar patrones inmutables |

### Mobile

- Ancho minimo soportado: **375px** (iPhone SE)
- Tablas se convierten en **cards** en mobile
- Sidebar se oculta como **offcanvas**
- Botones con texto largo se abrevian en mobile
- Kanban tiene flechas `<` `>` para mover tareas

### Componentes compartidos

## DEV Workarounds activos

<Warning>
  Los siguientes workarounds estan activos en el codigo fuente y deben removerse antes del deploy a produccion.
</Warning>

| Archivo | Workaround | Cuando remover |
|---------|------------|----------------|
| `_authenticated.tsx` | Default `userRole` a `'admin'` | Cuando el Custom Access Token Hook este habilitado en Supabase Dashboard |
| `auth-provider.tsx` | Fallback de `userRole` a `'admin'` | Mismo que arriba |
| `authorize()` SQL | Fallback a tabla `user_roles` | Funciona con o sin el hook; es permanente |

Busca los comentarios `// DEV fallback` en el codigo para encontrar estos workarounds.

## GitHub Actions CI

Configuracion recomendada en `.github/workflows/ci.yml`:

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npm run build
      - run: npx playwright install --with-deps chromium
      - run: npm run dev &
      - run: npx wait-on http://localhost:5173 --timeout 30000
      - run: npx playwright test
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: test-results
          path: test-results/
```

Este workflow ejecuta en cada push:
1. Build de produccion (incluye chequeo de tipos)
2. Instala Playwright con Chromium
3. Inicia el servidor de desarrollo
4. Ejecuta los 26 tests E2E
5. Sube screenshots como artefacto si algun test falla
