# Base de Datos

Impulsa CRM utiliza **Supabase** (PostgreSQL) como base de datos, con Row Level Security (RLS) en todas las tablas y Edge Functions para operaciones privilegiadas.

## Convenciones de base de datos clave

### Tipos de datos

| Convencion | Detalle |
|-----------|---------|
| **Dinero** | Siempre `NUMERIC(15,2)`, nunca `float` o `real` |
| **IDs** | `UUID` con `gen_random_uuid()` como default |
| **Timestamps** | `TIMESTAMPTZ` con `now()` como default |
| **Enums** | Definidos como `TEXT` con restriccion `CHECK` |
| **JSON** | `JSONB` para datos flexibles (items de factura, configuracion, respuestas) |
| **Campos calculados** | `GENERATED ALWAYS AS` para total_cost, neto, dias_al_pago, etc. |

### Row Level Security (RLS)

Todas las tablas tienen RLS habilitado. Las politicas usan la funcion `public.authorize()`:

```sql
CREATE POLICY "allow_all_for_authorized"
  ON public.tabla
  FOR ALL
  USING (public.authorize(ARRAY['admin', 'produccion', 'finanzas']))
  WITH CHECK (public.authorize(ARRAY['admin', 'produccion', 'finanzas']));
```

### Funcion `authorize()`

La funcion `authorize()` verifica el rol del usuario autenticado:

1. **Primer intento**: Lee el claim `user_role` del JWT (`auth.jwt() -> 'user_role'`)
2. **Fallback**: Si el claim no existe, busca en la tabla `user_roles`
3. **Comparacion**: Verifica si el rol esta en la lista de roles permitidos

```sql
CREATE OR REPLACE FUNCTION public.authorize(allowed_roles text[])
RETURNS boolean AS $$
DECLARE
  user_role text;
BEGIN
  -- Try JWT claim first
  user_role := (auth.jwt() -> 'user_role')::text;

  -- Fallback to user_roles table
  IF user_role IS NULL THEN
    SELECT role INTO user_role
    FROM public.user_roles
    WHERE user_id = auth.uid();
  END IF;

  RETURN user_role = ANY(allowed_roles);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

<Info>
  El fallback a la tabla `user_roles` permite que el sistema funcione sin el Custom Access Token Hook de Supabase habilitado (util en desarrollo).
</Info>

## Sistema de autenticacion

### Flujo de autenticacion

<Steps>
  <Step title="Login">
    El usuario ingresa email y contrasena. Supabase Auth valida y devuelve un JWT.
  </Step>
  <Step title="JWT con rol">
    El Custom Access Token Hook de Supabase inyecta el `user_role` como claim en el JWT.
  </Step>
  <Step title="Frontend">
    El `auth-provider.tsx` lee el rol del JWT y lo expone via Context a toda la app.
  </Step>
  <Step title="RLS">
    Cada query a Supabase incluye el JWT. Las politicas RLS verifican el rol usando `authorize()`.
  </Step>
</Steps>

### Roles y permisos

| Rol | Tablas con acceso completo | Restricciones |
|-----|---------------------------|---------------|
| `admin` | Todas | Ninguna |
| `produccion` | events, event_costs, event_providers, tasks, providers, clients, presupuestos, payments | Sin acceso a facturas, recibos, notas, settings, user_roles |
| `finanzas` | events, payments, facturas, recibos, notas, providers, clients, presupuestos | Sin acceso a settings, user_roles, workflows |

## Edge Functions

Las Edge Functions de Supabase se encuentran en `supabase/functions/` y manejan operaciones que requieren permisos elevados:

| Funcion | Descripcion |
|---------|-------------|
| `afip-emit` | Emision de facturas electronicas via AFIP |
| `create-user` | Creacion de usuarios con contrasena |
| `invite-user` | Invitacion de usuarios con link de configuracion |
| `manage-user` | Gestion de cuentas (activar/desactivar) |
| `mercadopago-checkout` | Creacion de preferencia de pago en MercadoPago |
| `mercadopago-webhook` | Recepcion de notificaciones de MercadoPago |
| `send-email` | Envio de emails transaccionales |
| `workflow-execute` | Ejecucion de pasos de workflow |

## Rutas publicas (sin RLS)

Algunas tablas tienen politicas especiales para acceso publico (sin autenticacion):

| Tabla | Politica publica | Uso |
|-------|-----------------|-----|
| `events` | SELECT donde `public_page_enabled = true` | Pagina publica del evento |
| `attendees` | SELECT/UPDATE via `qr_code_token` | Self check-in |
| `tickets` | SELECT donde `is_active = true` | Listado de entradas |
| `orders` | INSERT para crear ordenes | Compra de entradas |
| `survey_links` | SELECT/UPDATE via `token` | Encuestas publicas |
| `survey_responses` | INSERT | Envio de respuestas |
