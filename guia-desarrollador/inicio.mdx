# Inicio para Desarrolladores

## Vision general del proyecto

Impulsa CRM es una aplicacion single-page (SPA) construida con React y TypeScript, desplegada en Vercel, con Supabase como backend (base de datos PostgreSQL, autenticacion, storage y edge functions).

La interfaz de usuario esta completamente en **espanol** y la moneda es **ARS** (peso argentino).

## Primeros pasos

### Pre-requisitos

- **Node.js** >= 20
- **npm** >= 10
- Un proyecto de **Supabase** con las migraciones aplicadas
- Variables de entorno configuradas

### Instalacion

<Steps>
  <Step title="Clonar el repositorio">
    ```bash
    git clone https://github.com/tu-usuario/impulsa-crm.git
    cd impulsa-crm
    ```
  </Step>
  <Step title="Instalar dependencias">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Configurar variables de entorno">
    Crear un archivo `.env` en la raiz del proyecto:

    ```env
    VITE_SUPABASE_URL=https://tu-proyecto.supabase.co
    VITE_SUPABASE_ANON_KEY=tu-anon-key
    ```
  </Step>
  <Step title="Iniciar el servidor de desarrollo">
    ```bash
    npm run dev
    ```

    El servidor se inicia en `http://localhost:5173` (o el siguiente puerto disponible).
  </Step>
  <Step title="Acceder al sistema">
    Usa las credenciales de desarrollo:
    - **Email**: `admin@impulsa.com`
    - **Password**: `Impulsa2026!`
  </Step>
</Steps>

### Comandos disponibles

| Comando | Descripcion |
|---------|-------------|
| `npm run dev` | Inicia el servidor de desarrollo (Vite, puerto 5173+) |
| `npm run build` | Build de produccion (`tsc -b && vite build`) |
| `npm run lint` | Ejecuta ESLint sobre todo el proyecto |
| `npm run preview` | Preview del build de produccion |
| `npx tsc --noEmit` | Verifica tipos de TypeScript sin compilar |
| `npx playwright test` | Ejecuta los tests E2E (26 tests) |
| `npx playwright test --headed` | Ejecuta los tests con navegador visible |
| `npx playwright test --ui` | Abre el picker interactivo de tests |
| `npx supabase db push --linked` | Aplica migraciones pendientes en Supabase |

<Warning>
  Siempre ejecuta `npm run build` antes de hacer push. El build incluye chequeo de tipos de TypeScript y detecta errores que no se ven en desarrollo.
</Warning>
